# 共享上下文

本目录保留，但只作为跨角色共享资产区使用，不再承载任何单一角色的正式产物。

## 保留范围

`shared_context` 只允许存放以下三类公共资产：

- 共享标准
  命名规范、流程规则、术语约束、协作约定
- 共享场景
  多角色共用的测试场景、联调场景、样例场景说明
- 共享模板
  任务模板、评审模板、记录模板

## 不应放入的内容

以下内容不应继续放入本目录：

- 架构设计说明书、领域模型、演进方案
- Python 工程实现说明、任务拆解、实现摘要
- 对外交付文档、脚手架、demo、验收材料
- 临时验证产物、个人笔记、一次性草稿

这些内容应分别进入：

- `architect_agent/outputs/`
- `engineering_agent/inputs/` 或 `engineering_agent/outputs/`
- `execution_agent/outputs/`
- `logs/` 或对应角色目录下的 `tasks/notes`

## 当前推荐子目录

- `standards/`
  共享标准、术语、规则、口径
- `scenarios/`
  共享测试场景和联调场景说明
- `templates/`
  可复用的任务、评审、记录模板
- `references/`
  仅保留确实需要跨角色长期共享的参考资料

## 使用规则

1. 共享资产必须可被多个角色复用，不能只是某一轮任务的临时附件。
2. 如果一份文档已经成为某个角色的正式输出，应优先回到该角色目录，不在本目录保留主编辑副本。
3. 若共享内容停止维护或已被正式文档替代，应迁出或归档，不在本目录继续堆积。
4. 本目录不作为权威来源目录；涉及架构、工程、交付结论时，仍以上游正式文档为准。

## 当前状态

当前本目录按“保留但瘦身”原则管理：

- `templates/` 继续保留
- `standards/`、`scenarios/`、`references/` 仅作为共享占位目录，后续只放高复用公共材料
- 若后续长期为空，可继续保留目录结构，但不要人为补充低价值内容
