# 05 交付验收清单

## 1. 文档说明

本文是对外交付包中的交付验收清单，用于帮助外部团队和执行侧在正式交接前确认最基本的材料完整性和接入闭环。

说明：

- 本文属于执行侧验收材料，不替代架构侧放行策略
- 更上游的验证分层和放行口径以上游 `architect_agent/outputs/python-sdk-verification-and-acceptance.md` 为准
- 本文只保留对外交付时最小可核对的工程、联调、配置和文档项

## 2. 工程验收

- 已提供完整外部项目源码
- 已提供 `env.properties` 样例
- 已提供 `agent.properties` 样例
- 已提供启动脚本或启动说明
- 已提供最小可运行 Agent 模板

## 3. 联调验收

- 可以收到初始化命令
- 可以返回初始化响应
- 可以收到 Tick 命令或事件命令
- 可以返回处理响应
- 可以收到终止命令并完成退出

## 4. 配置验收

- 环境参数与业务参数边界清晰
- Agent 身份参数可独立替换
- 远程 YAML 地址可独立切换

## 5. 运行验收

- 启动日志完整
- 异常日志可定位
- 断连或错误情况下能给出清晰报错
- 业务逻辑与 SDK 框架逻辑未耦死在一起

## 6. 文档验收

- 业务方能按文档独立完成安装
- 业务方能按文档独立完成配置
- 业务方能按文档完成首轮联调
- 文档与当前 SDK 代码入口一致

## 7. 角色边界说明

在交付验收环节：

- `architect_agent` 负责定义更上游的验证策略和放行口径
- `engineering_agent` 负责补齐实现、验证摘要和必要样例
- `execution_agent` 负责组织对外交付材料、启动检查、联调样例和验收清单
