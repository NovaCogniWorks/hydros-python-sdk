## 当前示例

`examples/agents/` 下当前真实维护的示例是：

- [twins](../examples/agents/twins/)：孪生仿真示例。
- [ontology](../examples/agents/ontology/)：本体推理仿真示例。
- [centralscheduling](../examples/agents/centralscheduling/)：中央调度示例，复用 SDK 默认 MPC 路径。

查看真实示例列表：

```bash
python -m hydros_agent_sdk.launcher --launcher-dir examples -- --list
```

## 推荐学习路径

快速上手：

```text
1. 阅读根 README 的快速开始。
2. 复制 env.properties.example 为本地 env.properties。
3. 运行 launcher --list 查看真实示例。
4. 启动 twins 或 ontology。
5. 根据示例目录结构创建自己的 agent。
```

系统学习：

```text
1. 阅读文档中心。
2. 阅读新手智能体开发指南。
3. 查看智能体类型对比。
4. 阅读 examples/README.md。
5. 使用快速参考作为日常查询入口。
6. 修改 SDK 主链路后运行测试基线。
```

## 配置和依赖约定

真实 `env.properties` 是本地或部署配置，不作为文档示例提交；提交 `env.properties.example` 模板。

依赖按接入成本分层：

- `requirements.txt`：SDK 最小运行依赖。
- `requirements-dev.txt`：开发、构建、发布和 debug 工具。
- `requirements-examples.txt`：通用 examples 运行依赖。
- `requirements-pump.txt`：泵站调度业务依赖。
- `requirements-power.txt`：电站自定义 Agent 和 hydrosim 依赖。

## 维护规则

- 文档中的示例路径必须指向当前仓库真实存在的目录或文件。
- 新增示例后同步更新 [examples/README.md](../examples/README.md) 和 [docs/README.md](./README.md)。
- 新增依赖时优先放到对应分层 requirements 或 `pyproject.toml` optional dependency，不要把业务仿真栈混入 SDK 最小依赖。
- 新增真实环境配置项时同步更新 `env.properties.example`，不要提交真实 `env.properties`。
