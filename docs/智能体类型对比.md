# Hydros 智能体类型对比指南

## 📖 概述

Hydros Agent SDK 提供了多种智能体基类，每种基类适用于不同的仿真场景。本指南帮助你选择最适合你需求的智能体类型。

---

## 🏗️ 继承层次结构

```
BaseHydroAgent (抽象基类)
    ↓
TickableAgent (时间步驱动基类)
    ↓
    ├── OntologySimulationAgent (本体仿真)
    ├── TwinsSimulationAgent (孪生仿真)
    ├── CentralSchedulingAgent (中央调度)
    └── ModelCalculationAgent (模型计算)
```

---

| **驱动模式** | 时间步驱动 | 时间步驱动 | 时间步驱动 | 时间步驱动 |

## 1️⃣ TickableAgent - 通用时间步驱动智能体

### 📝 描述

最基础的时间步驱动智能体，提供了完整的生命周期管理和时间序列数据处理能力。适合作为所有自定义智能体的起点。

### ✅ 适用场景

- 通用仿真任务
- 自定义仿真逻辑
- 学习和原型开发
- 不确定使用哪种类型时的默认选择

### 🎯 核心特性

- ✅ 时间步驱动执行
- ✅ 时间序列数据缓存
- ✅ 边界条件处理
- ✅ MQTT 指标输出
- ✅ 完整的错误处理
- ✅ 灵活的扩展性

### 💻 代码示例

```python
from hydros_agent_sdk.agents import TickableAgent

class MyCustomAgent(TickableAgent):
    """自定义智能体"""

    def on_init(self, request):
        """初始化"""
        self.load_agent_configuration(request)
        # 你的初始化逻辑
        self.state_manager.init_task(self.context, [self])
        self.state_manager.add_local_agent(self)
        return SimTaskInitResponse(...)

    def on_tick_simulation(self, request):
        """执行仿真"""
        # 你的仿真逻辑
        results = self._my_simulation_logic(request.step)
        return self._convert_to_metrics(results)

    def on_terminate(self, request):
        """终止清理"""
        self.state_manager.terminate_task(self.context)
        self.state_manager.remove_local_agent(self)
        return SimTaskTerminateResponse(...)
```

### 📌 使用建议

- **推荐给新手**: 最简单、最灵活
- **快速原型**: 快速验证想法
- **自定义逻辑**: 完全控制仿真流程

### ⚠️ 注意事项

- 需要自己实现所有仿真逻辑
- 没有预定义的模型结构
- 需要手动处理拓扑（如果需要）

## 2️⃣ OntologySimulationAgent - 本体仿真智能体

### 📝 描述

基于本体和规则的仿真智能体，使用知识图谱和推理引擎进行仿真计算。适合基于专家知识和规则的调度决策。

### ✅ 适用场景

- 基于规则的水网调度
- 专家系统仿真
- 知识推理和决策
- 策略验证和优化
- 低精度快速仿真

### 🎯 核心特性

- ✅ 本体模型加载
- ✅ 规则引擎集成
- ✅ 前向/后向推理
- ✅ 知识图谱支持
- ✅ 规则文件配置
- ✅ 快速执行

### 💻 代码示例

```python
from hydros_agent_sdk.agents import OntologySimulationAgent

class MyOntologyAgent(OntologySimulationAgent):
    """本体仿真智能体"""

    def _initialize_ontology_model(self):
        """初始化本体模型"""
        # 创建规则引擎
        self._rule_engine = OntologyRuleEngine()

        # 加载本体
        self._rule_engine.load_ontology(self._topology)

        # 加载规则文件
        rule_file = self.properties.get_property('rule_file')
        if rule_file:
            self._rule_engine.load_rules(rule_file)

        # 配置推理参数
        reasoning_mode = self.properties.get_property('reasoning_mode', 'forward_chaining')
        self._rule_engine.set_reasoning_mode(reasoning_mode)

    def _execute_ontology_simulation(self, step: int):
        """执行本体推理"""
        # 收集边界条件
        boundary_conditions = self._collect_boundary_conditions(step)

        # 应用规则推理
        results = self._rule_engine.apply_rules(step, boundary_conditions)

        # 转换为指标
        return self._convert_results_to_metrics(results)
```

### 📋 配置示例

```yaml
# agent_config.yaml
reasoning_mode: forward_chaining  # 或 backward_chaining
rule_file: http://example.com/rules.yaml
inference_depth: 3
ontology_url: http://example.com/ontology.owl
```

### 📌 使用建议

- **规则驱动**: 调度逻辑可以用规则表达
- **专家知识**: 有明确的专家经验和规则
- **快速仿真**: 不需要高精度数值计算
- **策略验证**: 验证不同调度策略的效果

### ⚠️ 注意事项

- 需要定义本体模型和规则
- 推理结果依赖规则质量
- 不适合需要精确数值计算的场景
- 需要理解本体和推理概念

## 3️⃣ TwinsSimulationAgent - 数字孪生仿真智能体

### 📝 描述

高精度数字孪生仿真智能体，使用数值求解器进行精确的水力计算。适合需要高保真度仿真的场景。

### ✅ 适用场景

- 高精度水力仿真
- 数字孪生系统
- 实时预测和预警
- 精确的水力分析
- 与实际系统同步

### 🎯 核心特性

- ✅ 高精度数值求解
- ✅ 水力模型集成
- ✅ 实时边界条件同步
- ✅ 收敛性控制
- ✅ 详细的状态输出
- ✅ 模型校准支持

### 💻 代码示例

```python
from hydros_agent_sdk.agents import TwinsSimulationAgent

class MyTwinsAgent(TwinsSimulationAgent):
    """数字孪生智能体"""

    def _initialize_twins_model(self):
        """初始化孪生模型"""
        # 创建水力求解器
        self._hydraulic_solver = HydraulicSolver()

        # 初始化求解器
        self._hydraulic_solver.initialize(self._topology)

        # 配置求解器参数
        solver_params = {
            'time_step': self.properties.get_property('time_step', 60),
            'convergence_tolerance': self.properties.get_property('convergence_tolerance', 1e-6),
            'max_iterations': self.properties.get_property('max_iterations', 100),
            'solver_method': self.properties.get_property('solver_method', 'newton_raphson'),
        }
        self._hydraulic_solver.set_parameters(solver_params)

    def _execute_twins_simulation(self, step: int):
        """执行孪生仿真"""
        # 收集边界条件
        boundary_conditions = self._collect_boundary_conditions(step)

        # 执行水力求解
        results = self._hydraulic_solver.solve_step(step, boundary_conditions)

        # 检查收敛性
        if not results.converged:
            logger.warning(f"求解未收敛: step={step}, iterations={results.iterations}")

        # 转换为指标
        return self._convert_results_to_metrics(results)

    def on_boundary_condition_update(self, time_series_list):
        """处理实时边界条件更新"""
        # 更新求解器状态
        for time_series in time_series_list:
            self._hydraulic_solver.update_boundary_condition(
                time_series.object_id,
                time_series.metrics_code,
                time_series.time_series
            )
```

### 📋 配置示例

```yaml
# agent_config.yaml
time_step: 60  # 秒
convergence_tolerance: 0.000001
max_iterations: 100
solver_method: newton_raphson  # 或 gradient_descent
boundary_condition_metrics:
  - inflow
  - upstream_water_level
  - gate_opening
output_metrics:
  - water_level
  - flow_rate
  - velocity
  - pressure
```

### 📌 使用建议

- **高精度需求**: 需要精确的水力计算结果
- **实时系统**: 与实际系统保持同步
- **预测预警**: 提前预测系统状态
- **模型校准**: 需要与实测数据对比校准

### ⚠️ 注意事项

- 计算复杂度高，执行时间长
- 需要准确的拓扑和参数
- 对边界条件质量要求高
- 可能出现收敛性问题
- 需要水力学专业知识

## 4️⃣ CentralSchedulingAgent - 中央调度智能体

### 📝 描述

基于模型预测控制（MPC）的中央调度智能体，用于优化整个水网的调度策略。

### ✅ 适用场景

- 全局优化调度
- 多目标优化
- 预测性控制
- 能耗优化
- 成本最小化

### 🎯 核心特性

- ✅ MPC 优化算法
- ✅ 多目标优化
- ✅ 约束处理
- ✅ 滚动时域优化
- ✅ 预测模型集成
- ✅ 调度指令生成

### 💻 代码示例

```python
from hydros_agent_sdk.agents import CentralSchedulingAgent

class MyCentralSchedulingAgent(CentralSchedulingAgent):
    """中央调度智能体"""

    def _initialize_scheduling_model(self):
        """初始化调度模型"""
        # 创建 MPC 优化器
        self._mpc_optimizer = MPCOptimizer()

        # 配置优化参数
        optimization_params = {
            'prediction_horizon': self.properties.get_property('prediction_horizon', 24),
            'control_horizon': self.properties.get_property('control_horizon', 6),
            'objective_weights': self.properties.get_property('objective_weights', {
                'cost': 0.5,
                'energy': 0.3,
                'service': 0.2
            }),
        }
        self._mpc_optimizer.set_parameters(optimization_params)

        # 加载约束
        constraints = self._load_constraints()
        self._mpc_optimizer.set_constraints(constraints)

    def _execute_scheduling_optimization(self, step: int):
        """执行调度优化"""
        # 收集当前状态
        current_state = self._collect_current_state(step)

        # 获取预测数据
        predictions = self._get_predictions(step)

        # 执行 MPC 优化
        optimal_controls = self._mpc_optimizer.optimize(
            current_state=current_state,
            predictions=predictions,
            step=step
        )

        # 生成调度指令
        scheduling_commands = self._generate_scheduling_commands(optimal_controls)

        # 转换为指标
        return self._convert_to_metrics(scheduling_commands)
```

### 📋 配置示例

```yaml
# agent_config.yaml
prediction_horizon: 24  # 预测时域（小时）
control_horizon: 6      # 控制时域（小时）
optimization_interval: 3600  # 优化间隔（秒）

# 优化目标权重
objective_weights:
  cost: 0.5        # 成本最小化
  energy: 0.3      # 能耗最小化
  service: 0.2     # 服务质量最大化

# 约束条件
constraints:
  water_level_min: 2.0
  water_level_max: 10.0
  flow_rate_max: 100.0
  pump_switch_limit: 3  # 每天最多切换次数
```

### 📌 使用建议

- **全局优化**: 需要整体优化水网调度
- **多目标平衡**: 平衡成本、能耗、服务等多个目标
- **预测性控制**: 利用预测信息提前调度
- **复杂约束**: 有复杂的运行约束条件

### ⚠️ 注意事项

- 计算复杂度非常高
- 需要准确的预测模型
- 优化可能耗时较长
- 需要优化算法专业知识
- 可能出现无解或局部最优

## 5️⃣ ModelCalculationAgent - 模型计算智能体

### 📝 描述

事件驱动的模型计算智能体，响应特定事件进行计算，而不是按时间步执行。

### ✅ 适用场景

- 事件驱动计算
- 按需计算
- 异步处理
- 触发式分析
- 非周期性任务

### 🎯 核心特性

- ✅ 事件驱动
- ✅ 异步执行
- ✅ 按需计算
- ✅ 灵活触发
- ✅ 独立于时间步

### 💻 代码示例

```python
from hydros_agent_sdk.agents import ModelCalculationAgent

class MyModelCalculationAgent(ModelCalculationAgent):
    """模型计算智能体"""

    def on_event(self, event):
        """响应事件"""
        if event.event_type == "ALARM_TRIGGERED":
            # 执行报警分析
            analysis_result = self._analyze_alarm(event)
            return self._send_analysis_result(analysis_result)

        elif event.event_type == "DATA_QUALITY_CHECK":
            # 执行数据质量检查
            quality_report = self._check_data_quality(event)
            return self._send_quality_report(quality_report)
```

### 📌 使用建议

- **事件响应**: 需要响应特定事件
- **按需计算**: 不需要周期性执行
- **异步处理**: 独立于主仿真流程
- **辅助分析**: 提供辅助分析功能

### ⚠️ 注意事项

- 不适合周期性仿真
- 需要定义事件类型和处理逻辑
- 可能与时间步仿真不同步

## 🎯 选择决策树

```
开始
  ↓
是否需要周期性执行？
  ├─ 否 → ModelCalculationAgent (事件驱动)
  └─ 是 ↓
      是否需要高精度数值计算？
        ├─ 是 → TwinsSimulationAgent (数字孪生)
        └─ 否 ↓
            是否基于规则推理？
              ├─ 是 → OntologySimulationAgent (本体仿真)
              └─ 否 ↓
                  是否需要全局优化？
                    ├─ 是 → CentralSchedulingAgent (中央调度)
                    └─ 否 → TickableAgent (通用智能体)
```

## 💡 实际应用示例

### 场景 1: 城市供水调度系统

**需求**:
- 实时监控水网状态
- 优化泵站调度
- 降低能耗成本
- 保证供水质量

**推荐方案**:
```
TwinsSimulationAgent (实时状态监控)
    +
CentralSchedulingAgent (优化调度决策)
    +
OntologySimulationAgent (规则验证)
```

### 场景 2: 防洪预警系统

**需求**:
- 高精度水位预测
- 实时边界条件更新
- 快速响应能力

**推荐方案**:
```
TwinsSimulationAgent (高精度预测)
```

### 场景 3: 水质监测系统

**需求**:
- 事件触发分析
- 数据质量检查
- 异常检测

**推荐方案**:
```
ModelCalculationAgent (事件驱动分析)
```

### 场景 4: 智能灌溉系统

**需求**:
- 基于规则的灌溉决策
- 快速响应
- 简单易维护

**推荐方案**:
```
OntologySimulationAgent (规则驱动)
```

## 🔄 混合使用

在实际项目中，可以同时使用多种智能体类型：

```python
# 在同一个系统中运行多个智能体
callback = MultiAgentCallback(node_id="LOCAL")

# 注册孪生仿真智能体（高精度状态监控）
callback.register_agent_factory("TWINS_AGENT", twins_factory)

# 注册中央调度智能体（优化决策）
callback.register_agent_factory("SCHEDULING_AGENT", scheduling_factory)

# 注册本体仿真智能体（规则验证）
callback.register_agent_factory("ONTOLOGY_AGENT", ontology_factory)
```
