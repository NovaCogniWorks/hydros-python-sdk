## 🔌 连接问题

### 问题 1: 无法连接到 MQTT Broker

**症状**:
```
ERROR: Failed to connect to MQTT broker
ConnectionRefusedError: [Errno 61] Connection refused
```

**可能原因**:
1. MQTT Broker 未启动
2. IP 地址或端口配置错误
3. 网络不通
4. 防火墙阻止连接

**解决方案**:

```bash
# 1. 检查 MQTT Broker 是否运行
# 如果使用 Mosquitto:
systemctl status mosquitto
# 或
ps aux | grep mosquitto

# 2. 测试网络连接
ping 192.168.1.24
telnet 192.168.1.24 1883

# 3. 使用 MQTT 客户端测试
mosquitto_sub -h 192.168.1.24 -p 1883 -t "test" -v

# 4. 检查防火墙
# Linux:
sudo ufw status
sudo ufw allow 1883

# macOS:
sudo pfctl -s rules
```

**检查配置**:
```properties
# env.properties
mqtt_broker_url=tcp://192.168.1.24  # 确保 IP 正确
mqtt_broker_port=1883                # 确保端口正确
```

**症状**:
```
ERROR: MQTT authentication failed
Connection refused: Not authorized
```

**解决方案**:

```properties
# env.properties
# 取消注释并配置正确的用户名和密码
mqtt_username=your_username
mqtt_password=your_password
```

**验证认证**:
```bash
# 使用命令行测试认证
mosquitto_sub -h 192.168.1.24 -p 1883 \
  -u your_username -P your_password \
  -t "test" -v
```

### 问题 3: 连接成功但收不到消息

**症状**:
- 智能体启动成功
- 日志显示已连接
- 但没有收到任何协调命令

**可能原因**:
1. Topic 配置错误
2. 集群 ID 不匹配
3. 协调器未运行
4. 消息过滤问题

**解决方案**:

```bash
# 1. 监听所有 Topic，查看是否有消息
mosquitto_sub -h 192.168.1.24 -t "#" -v

# 2. 检查协调器发送的 Topic
# 应该是: /hydros/commands/coordination/{cluster_id}

# 3. 确认智能体订阅的 Topic
# 查看日志中的 "Subscribed to topic" 信息
```

**检查配置**:
```properties
# env.properties
# 确保集群 ID 与协调器一致
hydros_cluster_id=my_cluster

# 如果手动配置了 topic，确保格式正确
mqtt_topic=/hydros/commands/coordination/my_cluster
```

**启用调试日志**:
```python
# run.py
setup_logging(
    level=logging.DEBUG,  # 改为 DEBUG
    console=True
)
```

### 问题 7: 配置参数读取为 None

**症状**:
```python
value = self.properties.get_property('my_param')
# value 是 None，但配置文件中有这个参数
```

**可能原因**:
1. 配置文件格式错误
2. 参数名拼写错误
3. 未调用 `load_agent_configuration()`

**解决方案**:

```python
# 1. 确保先调用 load_agent_configuration
def on_init(self, request):
    self.load_agent_configuration(request)  # 必须先调用
    value = self.properties.get_property('my_param')

# 2. 使用默认值
value = self.properties.get_property('my_param', default='default_value')

# 3. 打印所有配置，检查是否加载成功
logger.info(f"所有配置: {self.properties._properties}")
```

**检查 YAML 格式**:
```yaml
# 正确格式
my_param: value

# 错误格式（缩进错误）
  my_param: value
```

## ⚙️ 仿真执行问题

### 问题 10: 仿真执行失败

**症状**:
```
ERROR: Simulation execution failure
SIMULATION_EXECUTION_FAILURE: Simulation failed at step 10
```

**诊断步骤**:

```python
def _execute_simulation(self, step, boundary_conditions):
    try:
        logger.info(f"开始执行仿真: step={step}")
        logger.debug(f"边界条件: {boundary_conditions}")

        # 你的仿真逻辑
        results = self._my_solver.solve(step, boundary_conditions)

        logger.info(f"仿真完成: {len(results)} 个结果")
        logger.debug(f"结果: {results}")

        return results

    except Exception as e:
        logger.error(f"仿真执行失败: {e}", exc_info=True)
        raise
```

**常见错误**:

1. **除零错误**:
```python
# 添加检查
if denominator == 0:
    logger.warning(f"除零错误: object_id={object_id}")
    denominator = 1e-10  # 使用小值代替
```

2. **数组越界**:
```python
# 添加边界检查
if index < len(array):
    value = array[index]
else:
    logger.warning(f"索引越界: index={index}, len={len(array)}")
    value = default_value
```

3. **空值处理**:
```python
# 检查空值
if value is None:
    logger.warning(f"值为空: object_id={object_id}, metrics={metrics_code}")
    value = 0.0  # 使用默认值
```

### 问题 11: 边界条件缺失

**症状**:
```
WARNING: Boundary condition missing for object 1001
```

**解决方案**:

```python
def _collect_boundary_conditions(self, step):
    boundary_conditions = {}
    missing_bc = []

    bc_metrics = ['inflow', 'water_level']

    if self._topology:
        for top_obj in self._topology.top_objects:
            for child in top_obj.children:
                object_bc = {}

                for metrics_code in bc_metrics:
                    value = self.get_time_series_value(
                        child.object_id,
                        metrics_code,
                        step
                    )

                    if value is not None:
                        object_bc[metrics_code] = value
                    else:
                        missing_bc.append(f"{child.object_name}.{metrics_code}")

                if object_bc:
                    boundary_conditions[child.object_id] = object_bc

    if missing_bc:
        logger.warning(f"缺失边界条件: {', '.join(missing_bc)}")

    return boundary_conditions
```

**使用默认值**:
```python
# 为缺失的边界条件提供默认值
value = self.get_time_series_value(object_id, metrics_code, step)
if value is None:
    value = self._get_default_boundary_condition(metrics_code)
    logger.debug(f"使用默认边界条件: {metrics_code}={value}")
```

## 🐌 性能问题

### 问题 14: 仿真执行缓慢

**症状**:
- 每个时间步耗时过长
- 协调器超时

**诊断步骤**:

```python
import time

def on_tick_simulation(self, request):
    start_time = time.time()

    # 分段计时
    t1 = time.time()
    boundary_conditions = self._collect_boundary_conditions(request.step)
    logger.debug(f"边界条件收集耗时: {time.time() - t1:.3f}s")

    t2 = time.time()
    results = self._execute_simulation(request.step, boundary_conditions)
    logger.debug(f"仿真执行耗时: {time.time() - t2:.3f}s")

    t3 = time.time()
    metrics_list = self._convert_results_to_metrics(results, request.step)
    logger.debug(f"指标转换耗时: {time.time() - t3:.3f}s")

    total_time = time.time() - start_time
    logger.info(f"总耗时: {total_time:.3f}s")

    return metrics_list
```

**优化建议**:

1. **使用缓存**:
```python
import functools

@functools.lru_cache(maxsize=128)
def _get_object_properties(self, object_id):
    return self._topology.get_object(object_id)
```

2. **批量处理**:
```python
# 不好: 逐个处理
for obj in objects:
    result = self._process_object(obj)
    self.send_metrics(result)

# 好: 批量处理
results = [self._process_object(obj) for obj in objects]
self.send_metrics_batch(results)
```

3. **避免重复计算**:
```python
# 预计算对象名称映射
self._object_names = {
    obj.object_id: obj.object_name
    for top_obj in self._topology.top_objects
    for obj in top_obj.children
}

# 使用缓存的映射
object_name = self._object_names.get(object_id, f"Object_{object_id}")
```

### 问题 15: 内存占用过高

**症状**:
- 内存持续增长
- 最终导致 OOM

**诊断步骤**:

```python
import psutil
import os

def _log_memory_usage(self):
    process = psutil.Process(os.getpid())
    mem_info = process.memory_info()
    logger.info(f"内存使用: {mem_info.rss / 1024 / 1024:.2f} MB")

# 在关键位置调用
def on_tick_simulation(self, request):
    self._log_memory_usage()
    # 你的代码
```

**解决方案**:

1. **清理缓存**:
```python
# 定期清理时间序列缓存
if request.step % 100 == 0:
    old_keys = [k for k, v in self._time_series_cache.items()
                if self._is_old_data(v)]
    for key in old_keys:
        del self._time_series_cache[key]
    logger.info(f"清理缓存: {len(old_keys)} 个条目")
```

2. **使用生成器**:
```python
# 不好: 一次性加载所有数据
all_results = [self._process(obj) for obj in objects]

# 好: 使用生成器
def _process_objects(self):
    for obj in self._topology.all_objects:
        yield self._process(obj)

for result in self._process_objects():
    # 处理结果
```

## 📋 故障排查清单

遇到问题时，按以下顺序检查：

- [ ] **连接**: MQTT Broker 是否运行？网络是否通？
- [ ] **配置**: agent.properties 和 env.properties 是否正确？
- [ ] **日志**: 启用 DEBUG 日志，查看详细信息
- [ ] **拓扑**: 拓扑文件是否可访问？格式是否正确？
- [ ] **边界条件**: 是否收到时间序列数据？
- [ ] **仿真逻辑**: 是否有异常？是否返回结果？
- [ ] **指标输出**: 是否生成指标？是否发送成功？
- [ ] **性能**: 是否有性能瓶颈？内存是否泄漏？
