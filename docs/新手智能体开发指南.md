# Hydros 智能体开发指南

## 📖 概述

本指南将帮助你快速上手 Hydros Agent SDK，开发自己的仿真智能体。通过学习本体仿真智能体（Ontology）和孪生仿真智能体（Twins）两个示例，你将掌握智能体开发的核心流程和最佳实践。

### 什么是 Hydros 智能体？

Hydros 智能体是分布式水网仿真系统中的独立计算单元，通过 MQTT 协议与协调器（Coordinator）通信，执行特定的仿真任务。每个智能体负责：
- 接收仿真任务初始化请求
- 按时间步执行仿真计算
- 处理边界条件更新
- 输出仿真指标数据
- 响应任务终止请求

### 5分钟创建第一个智能体

```python
from hydros_agent_sdk.agents import TickableAgent
from hydros_agent_sdk.protocol.commands import *
from hydros_agent_sdk.protocol.models import CommandStatus
from hydros_agent_sdk.utils.mqtt_metrics import create_mock_metrics

class MyFirstAgent(TickableAgent):
    """我的第一个智能体"""

    def on_init(self, request: SimTaskInitRequest) -> SimTaskInitResponse:
        """初始化智能体"""
        # 加载配置
        self.load_agent_configuration(request)

        # 注册到状态管理器
        self.state_manager.init_task(self.context, [self])
        self.state_manager.add_local_agent(self)

        return SimTaskInitResponse(
            context=self.context,
            command_id=request.command_id,
            command_status=CommandStatus.SUCCEED,
            source_agent_instance=self,
            broadcast=False
        )

    def on_tick_simulation(self, request: TickCmdRequest):
        """执行仿真步"""
        # 你的仿真逻辑
        metrics = [create_mock_metrics(
            source_id=self.agent_code,
            job_instance_id=self.biz_scene_instance_id,
            object_id=1001,
            object_name="Demo_Object",
            step_index=request.step,
            metrics_code="demo_value",
            value=42.0
        )]
        return metrics

    def on_terminate(self, request: SimTaskTerminateRequest) -> SimTaskTerminateResponse:
        """终止智能体"""
        self.state_manager.terminate_task(self.context)
        self.state_manager.remove_local_agent(self)

        return SimTaskTerminateResponse(
            context=self.context,
            command_id=request.command_id,
            command_status=CommandStatus.SUCCEED,
            source_agent_instance=self,
            broadcast=False
        )
```

## 🏗️ 核心概念

### 1. 智能体类型层次

```
BaseHydroAgent (基础抽象类)
    ↓
TickableAgent (时间步驱动基类)
    ↓
┌─────────────────────┬──────────────────────┬─────────────────────┐
│                     │                      │                     │
OntologySimulation  TwinsSimulation  CentralScheduling  ModelCalculation
Agent               Agent            Agent               Agent
(本体仿真)          (孪生仿真)       (中央调度)          (模型计算)
```

**选择合适的基类：**
- **TickableAgent**: 通用时间步驱动智能体（推荐新手使用）
- **OntologySimulationAgent**: 基于规则的本体推理仿真
- **TwinsSimulationAgent**: 高精度数字孪生仿真
- **CentralSchedulingAgent**: 中央调度优化（MPC）
- **ModelCalculationAgent**: 事件驱动的模型计算

### 2. 智能体生命周期

```
┌──────────────┐
│  协调器发送   │
│  初始化请求   │
└──────┬───────┘
       ↓
┌──────────────────────────────┐
│  on_init()                   │
│  - 加载配置                   │
│  - 加载拓扑                   │
│  - 初始化模型                 │
│  - 注册状态管理器             │
└──────┬───────────────────────┘
       ↓
┌──────────────────────────────┐
│  循环执行 (每个时间步)        │
│  ┌────────────────────────┐  │
│  │ on_tick_simulation()   │  │
│  │ - 收集边界条件          │  │
│  │ - 执行仿真计算          │  │
│  │ - 生成指标数据          │  │
│  │ - 发送MQTT指标          │  │
│  └────────────────────────┘  │
└──────┬───────────────────────┘
       ↓
┌──────────────────────────────┐
│  on_terminate()              │
│  - 清理资源                   │
│  - 注销状态管理器             │
└──────────────────────────────┘
```

### 步骤 2: 实现初始化方法

`on_init()` 是智能体的入口点，负责加载配置和初始化资源：

```python
from hydros_agent_sdk import ErrorCodes, AgentErrorContext
from hydros_agent_sdk.utils import HydroObjectUtilsV2

def on_init(self, request: SimTaskInitRequest) -> SimTaskInitResponse:
    """初始化智能体"""

    # 1. 加载智能体配置（从YAML）
    self.load_agent_configuration(request)

    # 2. 加载水网拓扑（可选）
    topology_url = self.properties.get_property('hydros_objects_modeling_url')
    if topology_url:
        with AgentErrorContext(ErrorCodes.TOPOLOGY_LOAD_FAILURE, self.agent_code) as ctx:
            self._topology = HydroObjectUtilsV2.build_waterway_topology(topology_url)

        if ctx.has_error:
            raise RuntimeError(f"拓扑加载失败: {ctx.error_message}")

    # 3. 初始化你的仿真模型
    self._initialize_my_model()

    # 4. 注册到状态管理器
    self.state_manager.init_task(self.context, [self])
    self.state_manager.add_local_agent(self)

    # 5. 返回成功响应
    return SimTaskInitResponse(
        context=self.context,
        command_id=request.command_id,
        command_status=CommandStatus.SUCCEED,
        source_agent_instance=self,
        broadcast=False
    )
```

### 步骤 3: 实现仿真执行方法

`on_tick_simulation()` 是核心仿真逻辑：

```python
from typing import List
from hydros_agent_sdk.utils.mqtt_metrics import MqttMetrics, create_mock_metrics

def on_tick_simulation(self, request: TickCmdRequest) -> List[MqttMetrics]:
    """执行仿真步"""
    step = request.step

    # 1. 收集边界条件
    boundary_conditions = self._collect_boundary_conditions(step)

    # 2. 执行仿真计算
    results = self._my_solver.solve(step, boundary_conditions)

    # 3. 转换结果为指标
    metrics_list = []
    for object_id, values in results.items():
        for metrics_code, value in values.items():
            metrics_list.append(create_mock_metrics(
                source_id=self.agent_code,
                job_instance_id=self.biz_scene_instance_id,
                object_id=object_id,
                object_name=f"Object_{object_id}",
                step_index=step,
                metrics_code=metrics_code,
                value=value
            ))

    return metrics_list

def _collect_boundary_conditions(self, step: int):
    """从时间序列缓存收集边界条件"""
    boundary_conditions = {}

    # 从配置获取需要的边界条件指标
    bc_metrics = self.properties.get_property(
        'boundary_condition_metrics',
        ['inflow', 'water_level']
    )

    # 遍历拓扑对象
    if self._topology:
        for top_obj in self._topology.top_objects:
            for child in top_obj.children:
                object_bc = {}
                for metrics_code in bc_metrics:
                    # 从缓存获取值
                    value = self.get_time_series_value(
                        child.object_id,
                        metrics_code,
                        step
                    )
                    if value is not None:
                        object_bc[metrics_code] = value

                if object_bc:
                    boundary_conditions[child.object_id] = object_bc

    return boundary_conditions
```

### 步骤 4: 实现终止方法

```python
def on_terminate(self, request: SimTaskTerminateRequest) -> SimTaskTerminateResponse:
    """终止智能体并清理资源"""

    # 1. 清理你的资源
    if self._my_solver:
        self._my_solver.cleanup()

    # 2. 注销状态管理器
    self.state_manager.terminate_task(self.context)
    self.state_manager.remove_local_agent(self)

    # 3. 返回成功响应
    return SimTaskTerminateResponse(
        context=self.context,
        command_id=request.command_id,
        command_status=CommandStatus.SUCCEED,
        source_agent_instance=self,
        broadcast=False
    )
```

```python
from hydros_agent_sdk.protocol.models import ObjectTimeSeries

def on_boundary_condition_update(self, time_series_list: List[ObjectTimeSeries]):
    """处理边界条件更新"""
    for time_series in time_series_list:
        logger.info(
            f"边界条件更新: object={time_series.object_name}, "
            f"metrics={time_series.metrics_code}, "
            f"values={len(time_series.time_series)}"
        )

        # 更新你的仿真状态
        if self._my_solver:
            self._my_solver.update_boundary_condition(
                time_series.object_id,
                time_series.metrics_code,
                time_series.time_series
            )
```

### 步骤 6: 创建启动脚本

```python
# run.py
import os
import time
import logging
from hydros_agent_sdk import (
    setup_logging,
    SimCoordinationClient,
    HydroAgentFactory,
    MultiAgentCallback,
    load_env_config,
)

# 导入你的智能体类
from my_agent import MyCustomAgent

# 配置日志
setup_logging(
    level=logging.INFO,
    hydros_cluster_id='my_cluster',
    hydros_node_id='LOCAL',
    console=True
)

logger = logging.getLogger(__name__)

def main():
    # 获取脚本目录
    script_dir = os.path.dirname(os.path.abspath(__file__))

    # 加载环境配置
    ENV_FILE = os.path.join(script_dir, "env.properties")
    env_config = load_env_config(ENV_FILE)

    BROKER_URL = env_config['mqtt_broker_url']
    BROKER_PORT = int(env_config['mqtt_broker_port'])
    TOPIC = env_config['mqtt_topic']

    # 智能体配置文件
    CONFIG_FILE = os.path.join(script_dir, "agent.properties")

    # 创建智能体工厂
    agent_factory = HydroAgentFactory(
        agent_class=MyCustomAgent,
        config_file=CONFIG_FILE,
        env_config=env_config
    )

    # 创建回调
    callback = MultiAgentCallback(node_id=os.getenv("HYDROS_NODE_ID", "LOCAL"))
    callback.register_agent_factory("MY_CUSTOM_AGENT", agent_factory)

    # 创建协调客户端
    sim_coordination_client = SimCoordinationClient(
        broker_url=BROKER_URL,
        broker_port=BROKER_PORT,
        topic=TOPIC,
        sim_coordination_callback=callback
    )

    callback.set_client(sim_coordination_client)

    # 启动服务
    try:
        logger.info("="*70)
        logger.info("启动自定义智能体服务")
        logger.info("="*70)
        logger.info(f"MQTT Broker: {BROKER_URL}:{BROKER_PORT}")
        logger.info(f"MQTT Topic: {TOPIC}")
        logger.info("="*70)

        sim_coordination_client.start()

        logger.info("服务启动成功！等待任务...")
        logger.info("按 Ctrl+C 停止服务")

        while True:
            time.sleep(1)

    except KeyboardInterrupt:
        logger.info("停止服务...")
        sim_coordination_client.stop()
        logger.info("服务已停止")

if __name__ == "__main__":
    main()
```

## 🛡️ 错误处理最佳实践

Hydros SDK 提供了完善的错误处理机制：

### 方式 1: 装饰器（推荐用于生命周期方法）

```python
from hydros_agent_sdk import handle_agent_errors, ErrorCodes

@handle_agent_errors(ErrorCodes.AGENT_INIT_FAILURE)
def on_init(self, request):
    # 任何异常都会被自动捕获并转换为错误响应
    self.load_agent_configuration(request)
    return SimTaskInitResponse(...)

@handle_agent_errors(ErrorCodes.AGENT_TICK_FAILURE)
def on_tick_simulation(self, request):
    # 自动错误处理
    return self._execute_simulation(request.step)
```

### 方式 2: 上下文管理器（推荐用于代码块）

```python
from hydros_agent_sdk import AgentErrorContext, ErrorCodes

with AgentErrorContext(ErrorCodes.TOPOLOGY_LOAD_FAILURE, self.agent_code) as ctx:
    topology = HydroObjectUtilsV2.build_waterway_topology(url)

if ctx.has_error:
    logger.error(f"拓扑加载失败: {ctx.error_message}")
    raise RuntimeError(ctx.error_message)
```

### 方式 3: 安全执行函数（推荐用于单个操作）

```python
from hydros_agent_sdk import safe_execute, ErrorCodes

success, result, error_msg = safe_execute(
    my_risky_function,
    ErrorCodes.SIMULATION_EXECUTION_FAILURE,
    self.agent_code,
    arg1, arg2
)

if not success:
    logger.error(f"执行失败: {error_msg}")
    return []
```

## 📊 完整示例对比

### 示例 1: 本体仿真智能体（规则驱动）

**适用场景**: 基于规则的水网调度、专家系统、知识推理

**核心特点**:
- 使用本体规则引擎
- 前向/后向推理
- 规则文件配置

```python
class MyOntologyAgent(OntologySimulationAgent):
    def _initialize_ontology_model(self):
        """初始化本体模型"""
        self._rule_engine = OntologyRuleEngine()
        self._rule_engine.load_ontology(self._topology)

        # 加载规则文件
        rule_file = self.properties.get_property('rule_file')
        if rule_file:
            self._rule_engine.load_rules(rule_file)

    def _execute_ontology_simulation(self, step: int):
        """执行本体推理"""
        boundary_conditions = self._collect_boundary_conditions(step)
        results = self._rule_engine.apply_rules(step, boundary_conditions)
        return self._convert_results_to_metrics(results)
```

### 示例 2: 孪生仿真智能体（高精度计算）

**适用场景**: 高精度水力仿真、数字孪生、实时预测

**核心特点**:
- 使用水力求解器
- 高精度数值计算
- 实时边界条件同步

```python
class MyTwinsAgent(TwinsSimulationAgent):
    def _initialize_twins_model(self):
        """初始化孪生模型"""
        self._hydraulic_solver = HydraulicSolver()
        self._hydraulic_solver.initialize(self._topology)

        # 配置求解器参数
        solver_params = {
            'time_step': self.properties.get_property('time_step', 60),
            'convergence_tolerance': 1e-6,
            'max_iterations': 100,
        }
        self._hydraulic_solver.set_parameters(solver_params)

    def _execute_twins_simulation(self, step: int):
        """执行水力仿真"""
        boundary_conditions = self._collect_boundary_conditions(step)
        results = self._hydraulic_solver.solve_step(step, boundary_conditions)
        return self._convert_results_to_metrics(results)
```

```python
def _process_topology_objects(self):
    for top_obj in self._topology.top_objects:
        for child in top_obj.children:
            yield child

# 分批处理
for obj in self._process_topology_objects():
    self._process_object(obj)
```
