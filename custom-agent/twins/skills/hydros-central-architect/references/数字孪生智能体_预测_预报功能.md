# 核心功能
hydros-engine

1, 基于 【京石段】进行水动力仿真，通过mcp获取水动力仿真数据；

2, 基于 【京石段】进行水动力仿真， 通过传感器模拟，增加噪声，收集噪声过程数据；

3, 基于 【京石段】进行水动力仿真， 通过传感器模拟，增加噪声，激活 渠池智能体的 ETL去噪功能，收集噪声过程数据和去噪的结果数据；

3，新建场景，增加【数字孪生智能体】，通过协调器同步协调，数字孪生智能体 在孪生复刻 本体的前提下，提供预测未来的功能；新增基于预测数据的预警功能；

## 数字孪生智能体
+ 接受中心coordinator协调，类似于本体仿真，按照步进进行计算，
+ 与本体仿真智能体 一步一仿真输出不同是，每次步进指令达到，例如第i步，则判断是否激活预测计算，激活条件暂定为i%10==0（滚动周期），需要预测未来 N步(预测步长)的计算结果，即第i步计算，输出Fi~Fi+N的预测数据；下一轮滚动预测计算激活，假定为第 j步，输出Fj ~ Fj+N的预测数据，需要进行数据拼接和覆盖，即j步输出后，预测数据的接收端缓存的数据 保留Fi步的数据，但更新最新的Fj ~ Fj+N与上一轮预测周期步数重合的数据，以第j步为准 ，数据不断叠加；
+ 类似于中央调度智能体，在接收到 水网工况事件相关的时间序列数据后，立即激活最新的预测滚动周期；
+ 预测数据的发送端是MQTT，接收端hydros-data通过MQTT订阅接收，数据缓存区在hydros-data的内存;
+ 每次数据更新后，激活 预测监控逻辑，根据预设的监控规则（<font style="color:#DF2A3F;">如何预设</font>），若有异常，则触发告警alert,并由hydros-data记录在数据库；
+ 在仿真任务完成后，则计算结果数据打包成csv，上传至rustfs

## Hydro-engine-skill
+ 上述流程是通过hydro-engine-mcp执行完成的
+ 升级hdyros-engine，在hydros-playground中的场景列表中，暂时屏蔽相关任务的replay功能
+ hydro-engine-skill在仿真接收到后，通过hydro-engine-mcp获取 本体输出数据/预测数据/默认时序数据/工况时序数据，保存在本地，这些数据共享了同一个任务上下文即（bizScenarioInstanceId）和 步数/步长等数据时间刻度；
+ hydros-skill需要具备绘图功能，将上述数据绘制成图表，并需要提供 总结报告模板，对数据的联动逻辑进行解读和展示

## 本体仿真智能体
+ 可以在mcp运行时，通过配置参数激活传感器模拟，注入噪声

## 渠池智能体
+ 可以在mcp运行时，通过配置参数激活ETL数据清洗功能
+ 在激活ETL噪声清洗逻辑的情况下，检查mcp仿真入参中的 是否保存原始清洗前的raw_data，若是，通过MQTT发送给hydros-data，数据缓存区在hydros-data的内存中，在任务结束后，数据打包成csv，上传至rustfs



## Station智能体
+ 为了更好地hydro-claw联动，在工况事件中扩展 电机指令事件，coordinator编排
+ 定向发送给 StationAgent， StationAgent将该工况转换成直接开度指令，下发给本体仿真智能体
